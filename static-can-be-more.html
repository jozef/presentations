Static Pages/Content

advantages

  • fast "processing" with web server
  • can be served by any web server (file://)
  • nginx - 10000 inactive HTTP keep-alive connections take about 2.5M memory
  • 100k request for .png (313kB) from 1000 clients
    in 12.5s (8034.86 [#/sec])
  • it's cheap to feed users with static pages
  • can be server through CDN

Static pages

disadvantages

  • there has to be some tool to generate them otherwise it will be insane to try to keep the same look
  • there is no "well known tool"
  • there is no "well known howto"

server side Scripting/Programming

advantages

  • that's the only way to get the credit card number
  • lot of books/tutorials/documentation
  • my friend/coworker/everyone does it

server side Scripting/Programming

disadvantages

  • cost CPU
  • cost RAM
  • mostly require specific web server (Apache HTTPD, IIS, Tomcat)
  • ASP, PHP, JSP, SSI, CGI, ...

Makefile + inotify

Template::Toolkit

  • http://search.cpan.org/perldoc?Template
  • This templating module allows adding special marks into the text that are processed. Default ones are [% %], but can be redefined.
  • Standard usage of Template Toolkit is for MVC frameworks, where it can be used as a View component to generate XML, plain text, ... and of course HTML.

ttree

  • TT has `ttree` command
  • that is made to convert a tree of templates
  • this way BA.pm.org is generated

_cfg.tt2,
_wrapper.tt2,
_header.tt2,
_footer.tt2,

redirect.tt2

events.xml + events-to-(html|rdf).xslt

  • events.xml + events-to-html.xslt => events.tt2
  • events.xml + events-to-rdf.xslt => news.rdf

who.xml + who-to-html.xslt + who-to-js.xslt

  • who.xml + who-to-html.xslt => who.tt2
  • who.xml + who-to-js.xslt => 06_mongers.js
